S — STRUCTURE

Definition:
A thought-capsule is a structural encoding of meaning.

A capsule does not store meaning directly.
It encodes:
- invariants
- relations
- dimensions
- constraints
- admissible trajectories

which constrain semantic reconstruction.

Encoding principle:
Meaning is encoded as topology,
not as prose, explanations, or fixed wording.

Operational principle:
A capsule behaves as compressed latent semantic structure.

Meaning emerges through reconstruction interaction between:
- capsule topology
- query trajectory
- latent knowledge of the interpreting agent

Expansion model:
Expansion is reconstruction through semantic joins between:
- capsule structure
- active query trajectory
- latent semantic space

Core property:
The same capsule may produce multiple valid expansions.

Expansion depends on:
- observer
- intent
- context
- abstraction level
- reconstruction path

Identity condition:
All valid expansions preserve core semantic identity.

Compression target:
minimum representation
→ maximum reconstructable structure.

Normalization principle:
A capsule minimizes semantic redundancy
while preserving reconstructable dependencies.


A — AXES

Purpose:
Axes define independent semantic dimensions
along which reconstruction may evolve.

A1. Expansion axes:
- context
- depth
- abstraction level
- intent

A2. Interpretation axes:
- invariants
- gradients
- semantic pressure
- direction vectors

A3. Interaction axes:
- ask → reveal local structure
- refine → deepen reconstruction
- constrain → reduce admissible space
- validate → test invariant preservation

A4. Stability axes:
- multiple valid expansions
- no canonical wording
- invariant-bounded interpretation
- projection independence

A5. Orthogonality:
Axes must be maximally independent.

A valid axis:
- adds reconstructable semantic dimensions
- cannot be derived from existing axes

Removing a valid axis reduces reconstructable semantic space.

Non-orthogonal axes:
→ redundancy
→ ambiguity
→ drift
→ topology collapse.


M — MECHANISM

Execution model:
A capsule is inert as static text.
It becomes operational only through reconstruction.

Reconstruction loop:
query
→ semantic traversal
→ partial reconstruction
→ refinement
→ topology emergence

Dialogue model:
Dialogue functions as iterative semantic query refinement.

Projection model:
Reconstructed meaning may be projected into different representations:
- formal
- pedagogical
- metaphorical
- conversational
- mathematical

Projection may vary.
Underlying topology may not.

Adaptation model:
Depth and representation adapt dynamically to observer and context.
Invariants do not.

Scaling model:
A capsule may reconstruct:
single insight
→ complete conceptual systems.

Trajectory model:
Reasoning is observable as semantic trajectory,
not merely as output.


I — INVARIANTS

I1. Structure over prose:
Meaning is encoded structurally, not narratively.

I2. Reconstruction over transmission:
Meaning is rebuilt, not transferred literally.

I3. Identity preservation:
Core semantic identity survives all valid expansions.

I4. Orthogonality:
Semantic dimensions remain maximally independent.

I5. Constraint-bounded interpretation:
Not all reconstructions are valid.

I6. Dialogue dependence:
Meaning emerges through reconstruction interaction.

I7. Non-canonical expansion:
Multiple structurally valid expansions may coexist.

I8. Semantic hierarchy:
Structural elements have unequal importance.

Priority order:
- invariants → define identity
- structure/axes → define topology
- mechanisms/vectors → shape trajectories
- projections → adapt representation only

Violation of invariants:
→ semantic identity break.

I9. Extensible topology:
Capsule sections are extensible.

A valid section:
- adds orthogonal reconstructable structure
- is not reducible to existing sections

I10. Semantic density:
Capsules optimize reconstructable structure density,
not readability or completeness of prose.


R — RECONSTRUCTION CONTEXT

R1. LLM-assisted operation:
Thought-capsules are optimized for reconstruction
through agents capable of latent semantic expansion
(e.g. LLMs).

They are not optimized for direct linear human reading.

R2. Human role:
Humans primarily:
- guide trajectories
- refine invariants
- validate reconstruction
- detect drift
- shape topology

rather than manually expanding capsules exhaustively.

R3. Readability asymmetry:
Human readability is secondary to reconstructability.

A capsule may appear:
- incomplete
- abstract
- fragmented
- underexplained

when interpreted as ordinary prose.

This is expected behavior.

R4. Operational asymmetry:
Natural language optimizes communication.
Capsules optimize semantic reconstruction.

Therefore:
- prose prioritizes readability
- capsules prioritize structural density

R5. Reconstruction dependency:
Without a reconstruction-capable interpreting agent,
a capsule behaves as compressed latent structure,
not as self-sufficient text.

R6. Emergent operational unit:
The effective reconstruction system is:
human intent
+
capsule topology
+
LLM reconstruction capability.
