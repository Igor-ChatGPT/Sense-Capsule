S — STRUCTURE

Definition:
A thought-capsule is a structural encoding of meaning, not a textual description.

Encoding principle:
Text represents topology (axes, invariants, relations), not content.

Reconstruction:
Meaning is reconstructed through dialogue (human or LLM), not read directly.

Core property:
The same capsule yields multiple valid expansions depending on query trajectory.

Invariant:
Core meaning is preserved across all valid reconstructions.

Compression target:
minimum text → maximum reconstructable structure.


A — AXES

A1. Expansion axes:
- context
- depth (level of detail)
- intent (why the question is asked)

A2. Interpretation axes:
- invariants (what cannot change)
- gradients (semantic transitions)
- direction vectors (where reasoning moves)

A3. Action axes:
- ask → reveal local structure
- refine → reveal deeper topology
- constrain → reduce expansion space

A4. Stability axes:
- no prescribed answers
- multiple valid expansions
- bounded by invariants

A5. Orthogonality (critical):
Axes must be maximally independent.
Each axis adds a new dimension of meaning, not a variation of another.

Non-orthogonal axes → redundancy → drift → loss of structure.


M — MECHANISM

Execution model:
A capsule is inert as static text; it becomes operational only in dialogue.

Reconstruction loop:
question → partial expansion → refinement → structure emergence

Control:
Expansion is driven by reader intent,
but constrained by invariants encoded in the capsule.

Adaptation:
Representation and depth adapt to the reader,
core meaning does not.

Scaling:
Expansion range: single statement → full conceptual system.


I — INVARIANTS

I1. Structure over content: meaning is encoded as topology, not prose.
I2. Reconstruction over transmission: meaning is rebuilt, not delivered.
I3. Invariance: core meaning survives all valid expansions.
I4. Orthogonality: axes are independent dimensions of meaning.
I5. Constraint-bounded interpretation: not all readings are valid.
I6. Dialogue-dependence: without interaction, capsule is incomplete.
I7. A capsule is stabilized through interaction with an agent capable of reconstructing meaning from structure (e.g., LLM), not authored in isolation.


B — BOUNDARIES

B1. Degeneration into text:
If capsule explains instead of encodes → loses reconstructability.

B2. Axis entanglement:
Non-orthogonal axes collapse dimensions → ambiguity increases.

B3. Missing invariants:
Without constraints → uncontrolled interpretation drift.

B4. Over-specification:
Too much detail → becomes documentation, not capsule.

B5. Under-specification:
Too little structure → cannot reconstruct meaning.


E — EVALUATION

Quality signals:

- Reconstruction diversity:
  Different readers produce different but consistent expansions.

- Invariant stability:
  Core meaning remains unchanged across expansions.

- Compression ratio:
  Information reconstructed >> information written.

- Drift resistance:
  Expansion does not diverge outside intended semantic space.

- Orthogonality score:
  Axes contribute independent explanatory power.


C — CONSEQUENCES

C1. Enables meaning compression beyond natural language limits.
C2. Decouples knowledge from specific wording.
C3. Makes reasoning observable as trajectory, not output.
C4. Allows LLMs to operate on structure instead of surface text.

Core claim:
Meaning should be encoded as structure that survives reconstruction, not as text that requires memorization.
